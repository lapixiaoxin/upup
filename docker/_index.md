# Docker 知识版块索引与维护说明

本目录用于沉淀 Docker 与容器化相关知识。内容应围绕企业开发、测试、交付和运维场景展开，避免停留在零散命令层面。

## 1. 模块定位

* **容器化基础**：理解镜像、容器、仓库、网络、卷、运行时等核心概念。
* **工程化构建**：沉淀 Dockerfile 规范、多阶段构建、镜像瘦身、缓存优化和安全扫描。
* **企业交付实践**：覆盖本地开发、测试环境、CI 构建、镜像仓库、发布回滚和故障排查。

## 2. 建议学习路径

1. 容器化解决了什么问题：环境一致性、依赖隔离、交付标准化。
2. Docker 核心对象：Image、Container、Volume、Network、Registry。
3. Dockerfile 与镜像构建规范。
4. Docker Compose 与本地多服务编排。
5. 企业镜像仓库、CI 构建、镜像版本管理。
6. 容器运行时排障、安全基线和资源限制。

## 4. 内容边界

* 本版块聚焦 Docker 与容器化基础，不直接展开 Kubernetes 的集群治理。
* Kubernetes、云原生平台、服务网格等主题后续可以独立成版块。
* 涉及部署时，优先说明 Docker 在交付链路中的位置，而不是泛讲运维平台。

## 5. 维护约束

* 示例命令需要标注运行环境和前置条件。
* Dockerfile 示例需要说明每一层设计目的，如缓存、权限、安全、体积和可复现性。
* 涉及生产环境时，需要覆盖日志、健康检查、资源限制、镜像来源和安全扫描。
* 新增文档后必须更新本索引的学习路径和文档清单。
