# Git 知识版块索引与维护说明

本目录用于沉淀真实企业环境中的 Git 协作实践。内容重点不是孤立命令速查，而是围绕团队协作、分支策略、提交规范、代码审查、版本发布、权限安全和故障恢复建立完整知识体系。

## 1. 模块定位

* **面向企业协作**：关注多人开发、代码审查、分支保护、发布节奏、权限审计等真实问题。
* **面向工程落地**：每个规范都应尽量给出工具链、命令、配置或流程示例。
* **面向长期维护**：文档需要能支持后续 GitHub、GitLab、CI/CD、Monorepo 等主题继续扩展。

## 4. 内容边界

* 本版块优先讨论 Git、GitHub、GitLab、代码托管平台、分支策略、提交规范、代码审查和版本历史管理。
* CI/CD 内容只在与 Git Hook、Webhook、分支保护、合并检查、发版触发相关时展开。
* Docker、Kubernetes、部署架构等内容应放入对应版块，只在 Git 流程有直接依赖时引用。

## 5. 维护约束

* 新增 Git 专题时，使用数字前缀命名，并在本索引中补充学习顺序。
* 命令示例必须说明使用场景，避免脱离上下文堆叠命令。
* 涉及团队规范时，需要说明适用团队规模、发布频率、自动化测试基础和协作成本。
* 涉及危险操作时，如 `reset`、`rebase`、强制推送，需要明确风险和替代方案。
